## Things I've built

**Smart Student Performance Analyser**
A prediction tool, not a toy demo. The frontend is a plain JS interface talking to a scikit-learn model behind the scenes, backed by a normalised SQL database with indexing so it doesn't choke as data grows. I kept ingestion, inference, and UI as separate modules on purpose — made it much easier to test and swap pieces independently.

**College Event Management System**
Handles registration, scheduling, and attendee tracking for college events. The interesting part was the frontend validation — catching bad input before it ever hits the server — paired with a schema built for a read-heavy workload (lots of people checking event details, comparatively few writes).

**Task Manager REST API** *(building now)*
FastAPI + SQLAlchemy. This one's about doing API design properly — clear endpoints, sensible resource structure — rather than just making something that technically responds to requests.
